# 参与改进

感谢你帮助改进 CTO Skill。这个项目最重要的目标是：让非技术用户容易理解、容易开始，并且不会漏掉真正关键的风险。

## 开始之前

- 错别字和小修正可以直接提交 Pull Request。
- 如果修改会改变提问顺序、Superpowers 的接力方式，或“什么时候停止并等待用户同意”，请先创建 Issue，说明真实使用场景和对用户的影响。
- 不要把 Grill With Docs、Superpowers 或其他第三方 Skill 的文件直接复制进本仓库。
- 新增内容前先问：它是否解决了真实问题？能否只在需要时出现？如果不能，就不要让默认流程变重。

## 本地检查

```bash
python3 scripts/validate_repo.py
```

如果修改会改变实际行为，请在 Pull Request 中说明：

- 修改前哪里不好用；
- 修改后用户会看到什么变化；
- 用过哪些脱敏后的真实场景；
- 是否改变手动调用、Superpowers 接力、返回 CTO Skill 或开发前停止规则。

## Pull Request 要求

- 保持 `ctoskill/SKILL.md` 简短；只在特定情况需要的内容放进 `references/`。
- 面向用户的文字使用日常语言。必须出现技术词时，紧接着解释它对产品的实际影响。
- 不提交密钥、个人信息、本机路径、内部对话或未脱敏的测试记录。
- 不把“批准设计”或“批准开发计划”理解成“已经批准开始开发”。
- 依赖关系有变化时，同步更新 README 和第三方声明。

提交贡献即表示你同意按照本项目的 MIT License 发布该贡献。
