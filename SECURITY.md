# Security policy

## Supported versions

当前仅维护最新发布版本。

## 报告安全问题

请不要在公开 Issue 中披露可被利用的安全问题、访问令牌或私人数据。

优先使用 GitHub 仓库 **Security** 页面中的 **Report a vulnerability** 私密报告入口。报告请包含：

- 受影响的文件或版本；
- 可复现步骤；
- 潜在影响；
- 建议修复方向（如有）。

## 安全边界

CTO Skill 是工作流说明文件，不包含后台服务，也不会自行上传项目内容。但它可以引导 Codex 读取和写入当前工作区，因此用户仍应审查项目权限、第三方 Skill 和计划执行范围。
