# 安全说明

## 当前维护版本

只维护最新发布版本。

## 怎样私密报告安全问题

不要在公开 Issue 中放入访问令牌、私人项目内容，或可以被他人直接利用的安全问题。

请使用 GitHub 仓库 **Security** 页面里的 **Report a vulnerability** 私密入口，并尽量提供：

- 哪个文件或版本受影响；
- 怎样重复看到这个问题；
- 可能造成什么影响；
- 如果你有建议，可以附上修复方向。

## 这个 Skill 会接触什么

CTO Skill 本身只是说明文件，没有自己的服务器，也不会自行上传项目内容。但运行它的 Agent 产品可能在你授权后读取或修改当前项目文件，也可能继续调用其他 Skill。

安装第三方 Skill 前，请检查来源、所需权限和是否包含可执行脚本。涉及删除文件、公开发布、付费服务或正式环境时，仍应看清 Agent 产品显示的授权范围再确认。
