# 风险触发路由

只在观察到下列触发条件时提出对应的短结果问题；没有触发条件意味着不展开专项审计，不生成强制报告模板或矩阵。

| 可观察触发条件 | 只问需要得到的结果 |
|---|---|
| 外部 AI、API 或平台能力 | Phase 0 的最小证据、可观察通过标准和失败路线是什么？ |
| 认证、多人或敏感数据 | 需要哪些安全与权限边界？ |
| 按量收费的付费服务 | 主要成本驱动是什么？ |
| 面向真实用户公开上线 | 需要哪些托管运维与恢复能力？ |
| 在中国大陆正式运营 | 大陆可用性与合规是否满足？ |
| 关键开源项目或高锁定依赖 | 许可证、维护、安全与退出路径是否可接受？ |
| 异步任务、支付、大文件或重要数据 | 失败、幂等与恢复如何处理？ |
