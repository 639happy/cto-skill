# 风险检查表

只检查项目已经遇到的情况。没有遇到就跳过，不生成额外报告。

| 项目里已经出现的情况 | 需要得到的简单答案 |
|---|---|
| 依赖外部 AI、接口或第三方平台 | 先做哪个最小验证？看到什么结果算通过？失败后换什么办法？ |
| 有登录、多人使用或敏感数据 | 谁能看见、修改和删除哪些数据？怎样避免看到别人的内容？ |
| 使用按量收费的服务 | 哪个用户动作最花钱？怎样设置额度、提醒或停止条件，避免意外费用？ |
| 要向真实用户公开上线 | 谁负责日常运行、故障提醒、备份和恢复？能否优先使用托管服务？ |
| 要在中国大陆正式运营 | 访问速度、服务可用性、数据和内容合规是否满足？ |
| 依赖关键开源项目或很难替换的服务 | 使用许可、维护情况、安全风险和以后替换的办法是否可以接受？ |
| 有后台长任务、支付、大文件或重要数据 | 失败后怎样继续？用户重复点击时怎样避免重复扣费、重复数据或文件丢失？ |
